# App 3：AI 校園心靈守護者 ── Scratch 專案藍圖

---

## 專案說明

本 Scratch 專案是「AI 校園心靈守護者」的**互動模擬版本**，用來展示 App 3 的核心邏輯：
感測器偵測學生情緒狀態 → AI 判斷風險等級 → 警示燈亮起 → 老師關懷卡片彈出。

雖然 Scratch 沒有真實感測器，但我用**鍵盤按鍵**模擬感測訊號（按 1 = 偵測到低風險、按 2 = 中風險、按 3 = 高風險），讓評審可以實際操作、看到完整流程。

---

## 角色清單

| 角色 | 功能說明 | 造型數量 |
|------|---------|---------|
| 守護者 AI 機器人 | 主角，負責分析風險、廣播指令 | 2 個（待機 / 分析中） |
| 學生角色 | 代表被偵測的學生，切換心情 | 3 個（開心 / 普通 / 難過） |
| 警示燈 | 顯示風險等級顏色 | 4 個（關閉 / 綠燈 / 黃燈 / 紅燈） |
| 關懷卡片 | 彈出並顯示 AI 關懷建議文字 | 1 個（可顯示/隱藏） |
| 老師角色 | 收到通知後走向學生 | 2 個（站立 / 行走） |
| 校園地圖背景 | 顯示感測節點位置 | 1 個背景 |

## 背景

### 背景 1：學校走廊 / 教室場景

- 畫面左側：教室門口（感測節點 A）
- 畫面中間：走廊（感測節點 B）
- 畫面右側：操場入口（感測節點 C）
- 各節點旁放置一顆小警示燈角色
- 畫面右上角有一個 AI 機器人（守護者）
- 畫面右下角有老師角色

## 積木腳本

---

### 守護者 AI 機器人的積木腳本

#### 腳本 1：初始化

```scratch
當 [綠旗] 被點擊
換成造型 [待機]
移到 x:(190) y:(130)
廣播 [系統啟動]
說 [AI 校園心靈守護者，啟動！正在監控校園...] (2) 秒
```

#### 腳本 2：偵測鍵盤輸入（模擬感測器訊號）

```scratch
當 [綠旗] 被點擊
重複無限次
  如果 <按鍵 [1 v] 被按下？> 那麼
    廣播 [低風險偵測] 並等待
  結束
  如果 <按鍵 [2 v] 被按下？> 那麼
    廣播 [中風險偵測] 並等待
  結束
  如果 <按鍵 [3 v] 被按下？> 那麼
    廣播 [高風險偵測] 並等待
  結束
  如果 <按鍵 [0 v] 被按下？> 那麼
    廣播 [恢復正常] 並等待
  結束
結束
```

#### 腳本 3：接收「低風險偵測」廣播 → 分析並廣播建議

```scratch
當接收到 [低風險偵測]
換成造型 [分析中]
說 [正在分析... 風險等級：低] (1.5) 秒
廣播 [顯示低風險警示]
廣播 [顯示關懷建議-低]
換成造型 [待機]
```

#### 腳本 4：接收「中風險偵測」廣播

```scratch
當接收到 [中風險偵測]
換成造型 [分析中]
說 [正在分析... 風險等級：中，建議老師主動關心] (2) 秒
廣播 [顯示中風險警示]
廣播 [顯示關懷建議-中]
廣播 [老師出動]
換成造型 [待機]
```

#### 腳本 5：接收「高風險偵測」廣播

```scratch
當接收到 [高風險偵測]
換成造型 [分析中]
說 [警告！偵測到高風險！立即通知老師！] (2) 秒
演奏聲音 [警示音效 v]
廣播 [顯示高風險警示]
廣播 [顯示關懷建議-高]
廣播 [老師出動]
換成造型 [待機]
```

#### 腳本 6：恢復正常

```scratch
當接收到 [恢復正常]
說 [偵測恢復正常，繼續監控。] (2) 秒
廣播 [關閉警示]
廣播 [隱藏關懷卡片]
```

### 學生角色的積木腳本

#### 腳本 1：初始化

```scratch
當 [綠旗] 被點擊
移到 x:(-50) y:(-80)
換成造型 [普通]
顯示
```

#### 腳本 2：自動隨機切換心情（模擬感測器持續偵測）

```scratch
當 [綠旗] 被點擊
重複無限次
  等待 (在 (3) 到 (8) 間隨機取數) 秒
  換成造型 (在 (1) 到 (3) 間隨機取數)
  -- 造型 1 = 開心，造型 2 = 普通，造型 3 = 難過
結束
```

> **說明**：這模擬感測器「持續偵測」的概念。
> 當造型切到「難過」時，就代表感測器偵測到異常訊號。
> 實際版本中，這個訊號來自真實硬體感測器。

#### 腳本 3：偵測自己的造型 → 觸發警示（模擬感測器邏輯）

```scratch
當 [綠旗] 被點擊
重複無限次
  如果 <(造型編號) = (3)> 那麼
    -- 難過造型 = 偵測到異常
    廣播 [中風險偵測]
    等待 (5) 秒  -- 避免重複廣播
  結束
結束
```

#### 腳本 4：高風險時學生顯示求救動作

```scratch
當接收到 [高風險偵測]
重複 (5) 次
  將 [顏色 v] 特效增加 (25)
  等待 (0.2) 秒
結束
清除所有圖形特效
```

#### 腳本 5：恢復正常

```scratch
當接收到 [恢復正常]
換成造型 [開心]
```

### 警示燈的積木腳本

> 每個感測節點放一個警示燈角色。以下為共用邏輯。

#### 腳本 1：初始化

```scratch
當 [綠旗] 被點擊
換成造型 [關閉]
顯示
```

#### 腳本 2：顯示低風險（黃燈）

```scratch
當接收到 [顯示低風險警示]
換成造型 [黃燈]
將 [亮度 v] 特效設定為 (0)
重複 (3) 次
  將 [亮度 v] 特效增加 (30)
  等待 (0.3) 秒
  將 [亮度 v] 特效增加 (-30)
  等待 (0.3) 秒
結束
```

#### 腳本 3：顯示中風險（橘色）

```scratch
當接收到 [顯示中風險警示]
換成造型 [橘燈]
-- 橘燈可用黃燈+顏色特效調整
將 [顏色 v] 特效設定為 (20)
重複 (5) 次
  將 [亮度 v] 特效增加 (25)
  等待 (0.2) 秒
  將 [亮度 v] 特效增加 (-25)
  等待 (0.2) 秒
結束
```

#### 腳本 4：顯示高風險（紅燈）

```scratch
當接收到 [顯示高風險警示]
換成造型 [紅燈]
清除所有圖形特效
重複 (10) 次
  將 [亮度 v] 特效增加 (40)
  等待 (0.15) 秒
  將 [亮度 v] 特效增加 (-40)
  等待 (0.15) 秒
結束
-- 保持紅燈亮著（不自動關閉）
```

#### 腳本 5：關閉警示

```scratch
當接收到 [關閉警示]
清除所有圖形特效
換成造型 [關閉]
```

### 關懷卡片的積木腳本

> 關懷卡片是一個寫有建議文字的圖片，平時隱藏，收到廣播後滑入畫面。

#### 腳本 1：初始化

```scratch
當 [綠旗] 被點擊
隱藏
移到 x:(0) y:(-200)  -- 藏在畫面下方
```

#### 腳本 2：顯示低風險建議

```scratch
當接收到 [顯示關懷建議-低]
換成造型 [低風險卡片]
-- 卡片上文字：「建議：多給予正向關注，若持續三天以上請主動問候。」
移到 x:(0) y:(-200)
顯示
重複直到 <(y 座標) > (-20)>
  將 y 座標增加 (15)
結束
等待 (5) 秒
廣播 [隱藏關懷卡片]
```

#### 腳本 3：顯示中風險建議

```scratch
當接收到 [顯示關懷建議-中]
換成造型 [中風險卡片]
-- 卡片上文字：「AI 建議：安排一對一談話，詢問近期睡眠與家庭狀況，必要時聯絡輔導室。」
移到 x:(0) y:(-200)
顯示
重複直到 <(y 座標) > (-20)>
  將 y 座標增加 (15)
結束
```

#### 腳本 4：顯示高風險建議

```scratch
當接收到 [顯示關懷建議-高]
換成造型 [高風險卡片]
-- 卡片上文字：「緊急！AI 建議立即介入！請聯絡輔導老師、家長，並依校安通報程序處理。」
移到 x:(0) y:(-200)
顯示
重複直到 <(y 座標) > (-20)>
  將 y 座標增加 (20)
結束
-- 高風險不自動消失，等老師按下「結案」按鈕
```

#### 腳本 5：隱藏卡片

```scratch
當接收到 [隱藏關懷卡片]
重複直到 <(y 座標) < (-190)>
  將 y 座標增加 (-15)
結束
隱藏
```

#### 腳本 6：點擊卡片 = 老師確認關懷（勾選清單）

```scratch
當角色被點擊
說 [老師已確認，正在執行關懷流程...] (2) 秒
廣播 [老師確認關懷]
廣播 [隱藏關懷卡片]
廣播 [恢復正常]
```

## 互動說明

### 操作方式（給評審的說明）

| 按鍵 | 模擬事件 |
|------|---------|
| 按 `0` | 系統恢復正常，關閉所有警示 |
| 按 `1` | 感測器偵測到低風險訊號（學生情緒有點低落）|
| 按 `2` | 感測器偵測到中風險訊號（需要老師主動關心）|
| 按 `3` | 感測器偵測到高風險訊號（緊急！老師立即介入）|
| 點關懷卡片 | 老師確認已執行關懷，標記結案 |

### 與真實 App 的對應關係

| Scratch 元件 | 真實 App 3 對應功能 |
|-------------|-------------------|
| 鍵盤按鍵 1/2/3 | Arduino R4 WiFi 感測節點 |
| 守護者 AI 機器人 | Gemini AI 分析模組 |
| 警示燈顏色切換 | 校園地圖上的節點警示指示 |
| 關懷卡片彈出 | AI 關懷建議面板 + 關懷清單 |
| 老師角色走向學生 | 老師收到推播通知後開啟 App |
| 點擊卡片確認 | 老師勾選清單 + 按「部署關懷」按鈕 |
| ALERT_SIGNAL | 高風險警示燈閃爍（紅燈腳本）|
| CARE_DEPLOYED | 關懷卡片被點擊後廣播結案 |
