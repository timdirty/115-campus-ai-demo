# App 1：AI 自動板擦機器人 ── 我的程式邏輯

---

## 我的程式是怎麼想的？

我做這個作品是因為我發現，老師每次下課都要自己擦黑板，又累又花時間。我想，如果有一個機器人可以幫老師擦黑板，老師就可以把時間用在更重要的事情上了！

我設計了一個 App，老師只要用平板點一下「要擦哪個區域」，App 就會透過電腦傳送指令給 Arduino 機器人，機器人收到指令之後就會自動移動到那個地方，用馬達帶動板擦來回移動，把字跡擦乾淨。擦完之後，機器人的 LED 燈還會顯示漂亮的慶祝圖案！

整個作品最厲害的地方是：App 有「AI 智慧辨識」功能，可以判斷哪個區域有字跡、選最有效率的路線，讓機器人不用繞遠路，節省時間和電量。

---

## 系統的三層架構

```
第一層：控制 App（老師的平板或電腦）
    └─ 老師用 App 點選要擦的區域和按鈕

第二層：Arduino 橋接（Node.js 程式，裝在電腦上）
    └─ 把 App 的按鈕指令翻譯成 Arduino 看得懂的命令
    └─ 透過 USB 傳給 Arduino

第三層：硬體機器人（Arduino UNO R4 WiFi）
    └─ 收到命令後，控制馬達讓板擦動起來
    └─ 同時用 LED 矩陣顯示狀態（笑臉、哭臉、煙火等）
```

---

## 主要邏輯（用小學生看得懂的方式寫）

### 一、連線部分

```
如果 老師點了「連線按鈕」：
    App 去找有沒有 Arduino 插在電腦上
    如果找到了：
        建立連線通道（速度：115200 baud）
        顯示「連線成功！」
        按鈕從灰色變成可以點的顏色
    如果沒找到：
        顯示「請先插上 USB 線」
        讓老師重新選擇連接埠

如果 老師點了「斷線按鈕」：
    把連線通道關閉
    所有按鈕變成灰色（不能點）
    顯示「已斷線」
```

---

### 二、清潔任務部分

```
如果 老師選了要擦的區域，然後按「開始清潔」：

    如果選的是「A區」：
        App 傳送「ERASE_REGION_A」指令給 Arduino
    如果選的是「B區」：
        App 傳送「ERASE_REGION_B」指令給 Arduino
    如果選的是「C區」：
        App 傳送「ERASE_REGION_C」指令給 Arduino
    如果選的是「全板擦除」：
        App 傳送「ERASE_ALL」指令給 Arduino

    接下來傳送「CLEAN_START」讓機器人開始動

    Arduino 收到後：
        啟動馬達，讓板擦開始移動
        LED 矩陣顯示「努力工作中」的圖案

    機器人擦拭過程：
        重複來回移動，直到指定區域擦完
        App 畫面上的任務日誌會顯示目前進度

    機器人擦完後：
        傳送「CLEAN_STOP」訊號告訴 App 完成了
        App 接收到後，記錄到任務日誌
        傳送「FIREWORK」指令讓 LED 矩陣顯示煙火
```

---

### 三、方向手動控制部分

```
如果 老師按了方向按鈕（↑↓←→）：
    把方向轉換成指令（UP、DOWN、LEFT、RIGHT）
    加上目前設定的速度（速度 1 到 5 格，可以用滑桿調整）
    傳送給 Arduino

    Arduino 收到後：
        按照方向和速度移動馬達
        一直到老師放開按鈕為止

如果 老師放開按鈕：
    傳送「停止」指令
    機器人停下來
```

---

### 四、暫停功能

```
如果 機器人正在清潔，老師按了「暫停」：
    App 傳送「PAUSE_TASK」指令
    Arduino 收到後，馬達立刻停止
    LED 矩陣顯示暫停符號
    App 顯示「任務暫停中」

如果 老師按下「繼續」：
    重新傳送「CLEAN_START」
    機器人繼續從剛才的位置擦下去
```

---

### 五、任務日誌

```
每當有重要事情發生，App 就把它記下來：
    - 連線時間
    - 開始哪個清潔任務的時間
    - 完成時間
    - 有沒有暫停過

這樣老師可以看到每次清潔的紀錄，
知道機器人一共幫忙擦了幾次黑板！
```

---

### 六、AI 智慧辨識部分（進階功能）

```
如果 AI 攝影機掃描到黑板：
    把黑板分成 A、B、C 三個區域
    檢查每個區域有沒有字跡

    如果 A區有字、B區沒字、C區有字：
        建議老師先擦 A區再擦 C區
        （這樣機器人不用跑來跑去，比較省時間）

    如果 全部都有字：
        建議直接用「ERASE_ALL 全板擦除」
        （一次擦完最快！）

    如果 全部都沒字：
        顯示「黑板已經很乾淨囉！」
        不需要清潔
```

## 我在這個作品裡學到了什麼

1. **Serial 通訊**：我學會了怎麼用 USB 讓電腦和 Arduino 說話。它們說話的方式是傳送英文指令（像 CLEAN_START、ERASE_ALL），Arduino 看到指令就知道要做什麼。

2. **前後端分工**：App 的畫面（按鈕、日誌）是「前端」，負責跟老師溝通；Node.js 橋接程式是「後端」，負責跟 Arduino 溝通。兩個部分各做各的事，不會混在一起。

3. **狀態管理**：機器人有很多狀態（待命、連線中、移動中、清潔中……），我要確保每個狀態只做對的事、不做錯的事。這讓我知道程式需要「想清楚狀態再寫」，不能想到哪寫到哪。

4. **除錯的重要性**：做這個作品的時候，馬達有時候收不到指令、LED 燈有時候不亮。我學會了用 Serial Monitor 看 Arduino 傳回來的訊息，一步一步找到問題在哪裡。這讓我覺得「找 bug」其實也很有趣，像在解謎一樣！
