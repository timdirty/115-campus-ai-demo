# App 2：校園服務機器人 ── 我的程式邏輯

## 我的程式是怎麼想的？

我做了一個管理校園服務機器人的 App！我覺得如果學校有機器人可以幫忙送東西，老師就不用一直跑來跑去，同學也可以更快收到需要的東西。

我的 App 可以讓老師或同學選擇用哪台機器人、要送到哪裡，然後可以在地圖上看機器人移動的狀況，最後機器人完成任務後，App 會自動產生一份報告。

我把整個程式邏輯想成四個大問題：
1. 怎麼知道機器人現在在做什麼？（儀表板）
2. 怎麼叫機器人去送東西？（派遣任務）
3. 怎麼知道機器人到了沒？（即時追蹤）
4. 任務完成後要記錄下來（完成回報）

---

## 系統架構

我的 App 分成這幾個部分：

```
使用者（老師/同學）
       |
       | 打開 App
       v
  +-----------+
  |   儀表板   |  <-- 首頁，看到所有機器人的狀態
  +-----------+
       |
   點選功能
   /    |    \
  /     |     \
送貨   地圖   排程
追蹤   派遣   任務
  \     |     /
   \    |    /
    任務完成回報
```

最重要的部分是「儀表板」，就像飛機的駕駛艙，所有資訊都在這裡看。

---

## 主要邏輯

### 儀表板邏輯

```
每一秒更新一次畫面：

  對每台機器人（1號、2號、3號、4號）：
    顯示它的名字
    顯示它的電量（用進度條表示）
    顯示它目前在做什麼（待命/移動中/完成）
    顯示它的速度

  顯示今天總共派了幾個任務
  顯示已經完成幾個任務
  
  如果有任務在進行中：
    顯示目前在第幾步（共5步）
    第1步：派遣
    第2步：移動
    第3步：交付
    第4步：回報
    第5步：完成
```

我覺得用 5 個步驟表示進度最清楚，就像遊戲裡的任務進度條一樣！

---

### 派遣任務

```
使用者按下「派遣」按鈕：

  第一步：檢查機器人狀態
    如果 機器人電量 < 20%：
      顯示警告：「電量不足，請先充電」
      停止派遣
    如果 機器人狀態 不是「待命」：
      顯示提醒：「這台機器人還在執行任務」
      停止派遣

  第二步：確認目的地
    取得使用者在地圖上點選的位置
    計算從現在位置到目的地的路線
    如果 目的地無法到達（例如障礙物擋住）：
      顯示提醒：「這個地方無法到達，請換一個」
      停止派遣

  第三步：發送任務
    把任務資料整理好：
      - 哪台機器人：機器人編號
      - 去哪裡：目的地名稱 + 座標
      - 什麼時間：現在的時間
      - 是誰派的：使用者名稱
    
    傳送任務給機器人
    更新儀表板上的任務數量（加 1）
    
    告訴使用者：「任務已成功派遣！」
```

我覺得最重要的是要先「檢查」再「派遣」，這樣才不會白跑一趟。就像出門前要確認鑰匙帶了沒有！

---

### 追蹤機器人

```
當任務派出去之後：

  一直重複做這件事（每 0.5 秒更新一次）：

    從機器人取得最新位置（x座標、y座標）
    在地圖上把機器人的圖示移到新位置
    更新電量顯示
    更新速度顯示

    如果 機器人位置 接近 目的地（距離 < 10 格）：
      顯示：「機器人快到了！」
      播放提示音效

    如果 機器人到達目的地：
      停止追蹤
      進入「交付確認」流程

  追蹤地圖會顯示：
    - 機器人目前位置（用小機器人圖示）
    - 目的地（用旗子圖示）
    - 移動的路線（用虛線表示）
```

我用虛線表示路線，這樣老師就可以看到機器人走了哪條路，很直覺！

---

### 完成回報

```
當機器人完成任務：

  第一步：等待接收方確認
    在畫面上顯示確認按鈕
    等待對方按「我收到了」
    
    如果 等了超過 60 秒 還沒人確認：
      自動發提醒通知給老師

  第二步：生成任務報告
    整理這次任務的所有資料：
      任務編號：（自動產生）
      機器人號碼：（哪台做的）
      派遣時間：（任務開始時間）
      完成時間：（現在時間）
      花了多久：（完成時間 - 派遣時間）
      目的地：（送到哪裡）
      電量消耗：（執行前電量 - 執行後電量）
    
    把報告顯示在「學生回報」頁面
    把報告存到任務記錄

  第三步：更新儀表板
    已完成任務數量 加 1
    機器人狀態改回「待命」
    在儀表板顯示：「任務完成！」
    放一個打勾的動畫
```

我喜歡把任務記錄存下來，這樣可以統計一個月做了幾個任務，證明機器人真的很有用！

---

### 排程任務（進階功能）

```
如果老師想要設定「明天早上 8 點送東西」：

  填入資料：
    時間：（年/月/日 時:分）
    機器人：（選哪台）
    目的地：（要送去哪）
    備註：（要送什麼東西）

  系統儲存這個排程

  到了設定時間的時候：
    自動檢查機器人狀態
    如果機器人可以派遣：
      自動執行派遣任務的流程
    如果機器人還在忙：
      等到機器人空下來再執行
      或是改用另一台機器人
```

排程功能就像鬧鐘一樣，設好之後不用一直記著，系統會自己在對的時間做！

## 我的程式用了哪些技術？

（這裡是給評審老師看的）

| 技術名稱 | 我用在哪裡 | 為什麼用這個 |
|----------|------------|--------------|
| React | 整個 App 的畫面 | 讓畫面更新很快，不用整頁重新載入 |
| 狀態管理 | 機器人的電量、位置、狀態 | 讓多個地方都可以看到最新資料 |
| 計時器 | 每 0.5 秒更新一次追蹤地圖 | 模擬即時更新的效果 |
| 條件判斷 | 電量檢查、到達偵測 | 讓機器人做出正確的決定 |
| 資料儲存 | 任務記錄、排程清單 | 讓資料不會不見 |

## 我學到了什麼

做這個 App 讓我學到了好多東西！

**程式設計方面：**
- 我學會了「狀態機」的概念，就是一個機器人在不同時間會有不同的狀態，而且狀態之間要有規則才能互相切換。就像紅綠燈，不能直接從紅燈跳到綠燈，中間要經過黃燈！
- 我學會了用「迴圈」讓程式一直重複做某件事，像追蹤機器人位置就是一直重複更新。
- 我學會了「條件判斷」很重要，要先檢查才能做，不然會出錯。

**解決問題方面：**
- 一開始我不知道怎麼在地圖上顯示機器人位置，後來我想到用 x、y 座標來表示，就像數學課學的坐標一樣！
- 電量不足的問題讓我學到要在做事情之前先「檢查條件」，這在程式設計裡叫做「防呆設計」。

**對機器人的想像：**
- 如果真的有校園服務機器人，我覺得可以幫忙送午餐、送文件、送醫藥給保健室，甚至可以在下雨天幫同學送傘！
- 機器人不是要取代人，是要幫忙做那些「麻煩但不危險」的小事，讓老師有更多時間教我們。

**最印象深刻的：**
我覺得最難但也最有成就感的是「即時追蹤地圖」。要讓一個小圖示在地圖上動來動去，需要計算座標、計算距離、計算方向，用到了很多數學！我以為數學只有在考試的時候才用得到，沒想到做程式也很需要！

---

## 給評審的話

我們這組做校園服務機器人，是因為我們觀察到學校裡很多「跑腿」的事情，像是老師要送聯絡簿、同學要去保健室拿藥，這些事情雖然小，但是加起來花了很多時間。

如果有一個 App 可以管理機器人幫大家做這些事，學校就可以更有效率，老師和同學也可以專心在更重要的事情上。

我們設計的 App 考慮到了很多實際的狀況，像是電量不足怎麼辦、機器人路上卡住怎麼辦、怎麼留下記錄等等。我們希望這不只是一個「看起來很酷」的 App，而是真的可以在學校用的工具！
