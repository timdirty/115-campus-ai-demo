# App 2：校園服務機器人 ── Scratch 專案藍圖

## 專案說明

這個 Scratch 專案模擬我們做的「校園服務機器人 App」，用積木程式來展示機器人怎麼接受任務、移動到目的地、完成交付、回報結果的整個流程。老師或同學可以按下綠旗子開始，點選不同機器人，指定送貨地點，看機器人一步一步完成任務！

## 專案連結

> Scratch 帳號：校園 AI 競賽組  
> 專案名稱：校園服務機器人 v1.0  
> （請在 scratch.mit.edu 用學校帳號登入後查看）

---

## 角色（Sprites）清單

| 編號 | 角色名稱 | 說明 |
|------|----------|------|
| 1 | 服務機器人 | 主角！有 4 個造型對應 1～4 號機器人 |
| 2 | 校園地圖 | 顯示各教室、圖書館、保健室位置 |
| 3 | 任務卡片 | 浮在畫面上顯示目前任務文字 |
| 4 | 完成標誌 | 任務完成時出現的大打勾圖示 |

---

## 背景

### 背景 1：學校平面圖
- 畫出學校的俯視圖（上面看下去）
- 標示地點：教室 A、教室 B、圖書館、保健室、操場、廁所
- 顏色：走廊用淺灰色，教室用淺黃色，戶外用淺綠色
- 每個地點旁邊有文字標示

## 積木腳本（每個角色）

---

### 角色 1：服務機器人的積木腳本

#### 造型說明
- 造型 1：1 號機器人（藍色，貼「1」號碼牌）
- 造型 2：2 號機器人（橘色，貼「2」號碼牌）
- 造型 3：3 號機器人（綠色，貼「3」號碼牌）
- 造型 4：4 號機器人（紫色，貼「4」號碼牌）

---

#### 腳本 A：初始化（綠旗啟動）

🟡 **事件觸發：點綠旗**
```
當 [綠旗] 被點擊
  換成造型 [1號機器人]
  移到 x: (0) y: (-100)  // 回到學校中央
  面朝 (90 度)           // 面朝右邊
  顯示
  將 [目前狀態] 設為 [待命]
  將 [電量] 設為 [100]
  將 [任務數] 設為 [0]
  將 [已完成] 設為 [0]
  廣播 [更新儀表板]
  說 [我準備好了！請派遣任務] 2 秒
```

---

#### 腳本 B：選擇機器人編號

🟢 **事件觸發：接收到「選擇機器人」廣播**
```
當收到 [選擇機器人]
  如果 <(選擇的號碼) = [1]> 那麼
    換成造型 [1號機器人]
  否則如果 <(選擇的號碼) = [2]> 那麼
    換成造型 [2號機器人]
  否則如果 <(選擇的號碼) = [3]> 那麼
    換成造型 [3號機器人]
  否則
    換成造型 [4號機器人]
  說 (連接 [我是] (選擇的號碼) [號機器人！]) 2 秒
```

---

#### 腳本 C：接收派遣任務

🔵 **事件觸發：接收到「派遣任務」廣播**
```
當收到 [派遣任務]
  將 [目前狀態] 設為 [接收任務中]
  廣播 [更新儀表板]
  說 [收到任務！確認目的地中...] 2 秒

  如果 <(電量) < [20]> 那麼
    說 [電量不足！需要先充電] 3 秒
    將 [目前狀態] 設為 [充電中]
    廣播 [更新儀表板]
    等待 (3) 秒        // 模擬充電時間
    將 [電量] 設為 [100]
    說 [充電完成！繼續執行任務] 2 秒

  將 [目前狀態] 設為 [前往目的地]
  廣播 [更新儀表板]
  廣播 [開始移動]
```

---

#### 腳本 D：移動到目的地

🟠 **事件觸發：接收到「開始移動」廣播**
```
當收到 [開始移動]
  說 [正在前往目的地...] 1 秒

  // 依據目的地設定終點座標
  如果 <(目的地) = [圖書館]> 那麼
    將 [終點X] 設為 [-150]
    將 [終點Y] 設為 [80]
  否則如果 <(目的地) = [保健室]> 那麼
    將 [終點X] 設為 [150]
    將 [終點Y] 設為 [80]
  否則如果 <(目的地) = [教室A]> 那麼
    將 [終點X] 設為 [-150]
    將 [終點Y] 設為 [-50]
  否則
    將 [終點X] 設為 [150]
    將 [終點Y] 設為 [-50]

  // 慢慢移動過去（讓觀眾看得到）
  重複執行 直到 <碰到邊緣?> 或 <(到達目的地) = [是]>
    面朝 (終點X) (終點Y) 方向
    移動 (5) 步
    將 [電量] 增加 (-1)   // 移動會消耗電量
    等待 (0.05) 秒

    // 判斷有沒有到達
    如果 <(與終點距離) < [20]> 那麼
      將 [到達目的地] 設為 [是]

  廣播 [到達目的地]
```

---

#### 腳本 E：到達目的地、完成交付

🟢 **事件觸發：接收到「到達目的地」廣播**
```
當收到 [到達目的地]
  將 [目前狀態] 設為 [到達交付]
  廣播 [更新儀表板]
  說 [已到達！等待確認收取...] 2 秒

  等待 (2) 秒              // 模擬對方確認收取

  說 [對方確認收到！準備回報] 2 秒
  將 [目前狀態] 設為 [回傳回報]
  廣播 [更新儀表板]
  等待 (1) 秒

  廣播 [任務完成]
```

---

#### 腳本 F：任務完成

🟢 **事件觸發：接收到「任務完成」廣播**
```
當收到 [任務完成]
  將 [目前狀態] 設為 [任務完成]
  將 [已完成] 增加 (1)
  廣播 [更新儀表板]
  說 [任務完成！正在回傳報告] 2 秒

  // 播放完成音效
  播放音效 [叮咚] 直到播放完畢

  // 回到起始位置（待命狀態）
  等待 (2) 秒
  在 (1) 秒內滑行到 x: (0) y: (-100)
  將 [目前狀態] 設為 [待命]
  廣播 [更新儀表板]
  廣播 [顯示完成標誌]
  說 [回到待命，等候下一個任務！] 2 秒
```

## 變數清單

| 變數名稱 | 用途 | 初始值 |
|----------|------|--------|
| 選擇的號碼 | 目前選中的機器人編號（1～4） | 1 |
| 目的地 | 要送達的地點名稱 | 空白 |
| 目前狀態 | 機器人當前狀態文字 | 待命 |
| 電量 | 機器人電量（0～100） | 100 |
| 任務數 | 總共接到幾個任務 | 0 |
| 已完成 | 已完成的任務數 | 0 |
| 終點X | 目的地的 X 座標 | 0 |
| 終點Y | 目的地的 Y 座標 | 0 |
| 到達目的地 | 是否抵達（是/否） | 否 |
| 與終點距離 | 用來判斷到達的距離值 | 0 |

## 互動說明

1. **按下綠旗**：整個系統初始化，機器人回到學校中央待命。
2. **點擊機器人**：彈出詢問框，輸入想選的號碼（1～4）。
3. **輸入目的地**：在另一個詢問框輸入目的地（圖書館/保健室/教室A/教室B）。
4. **廣播觸發**：程式自動廣播「派遣任務」，機器人開始動作。
5. **看機器人移動**：機器人從中央慢慢滑行到目的地。
6. **自動完成**：到達後等待 2 秒（模擬確認），然後回報完成，回到待命。
7. **計數更新**：「已完成」變數自動加 1，顯示在畫面角落。

> 小提示：可以把「電量」變數顯示在畫面上，這樣觀眾可以看到機器人移動時電量慢慢減少，更有真實感！
