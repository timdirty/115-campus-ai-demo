# 學生操作與報告講解稿：AI 校園心靈守護者

## 一句話介紹

我們這組做的是「匿名、非診斷的校園關懷支援系統」。它不是判斷誰有病，而是幫老師和輔導室更早看到需要關心的提醒，並提供自我照護、聊天支持和節點監控。

## 學生操作入口

- 公開展示網址：`https://timdirty.github.io/115-campus-ai-demo/app3/`
- 本機開發版：執行 `npm run dev` 後開啟 Vite 顯示的網址。
- 沒有 Firebase、Gemini 或 Arduino 時也能完整操作，資料會存在瀏覽器 localStorage。

## 上台分工建議

- 第一位同學：介紹為什麼校園需要早期關懷。
- 第二位同學：操作總覽與預警中心。
- 第三位同學：操作自我照護、匿名心情牆與聊天支持。
- 第四位同學：操作節點監控、硬體提示紀錄、匯出或重置。

## 3 分鐘講解稿

大家好，我們是 AI 校園心靈守護者小組。我們的作品不是心理診斷工具，也不收集真實學生個資，而是用匿名資料協助老師更早發現需要關心的情況。

首先在總覽頁，可以看到校園穩定度、待關懷提醒和節點狀態。這些資料都是示範代號，重點是呈現「匿名、非診斷、可持續追蹤」。

接著進入預警中心。我們選一筆提醒，勾選處置清單，例如「導師已確認」、「安排關懷談話」或「轉介輔導室」。按下佈署關懷後，系統會新增支持方案，也會留下硬體提示紀錄，未來可以接到 UNO R4 的燈號或提示器。

第三步是自我照護。學生可以做心情簽到，也可以在匿名心情牆發表一片葉子，其他人能給支持。這讓學生不需要公開身份，也能表達目前感受。

第四步是安全空間聊天。我們輸入考試壓力或人際困擾，系統會用支持性的語氣回覆，提醒使用者可以找可信任的大人或輔導室。它不會說自己在做診斷。

最後到節點頁，可以查看校園不同位置的節點狀態，也能重新連線離線節點。整個流程從提醒、處理、自我照護、聊天支持到節點監控，形成一個完整的校園關懷閉環。

## 操作步驟卡

1. 開啟總覽頁，先說明匿名、非診斷、關懷支援。
2. 進入預警中心，選一筆提醒。
3. 勾選處置清單，按下佈署關懷。
4. 回總覽確認硬體提示紀錄或支持方案增加。
5. 到自我照護做一次心情簽到。
6. 在匿名心情牆新增一片葉子，或按支持。
7. 到安全空間聊天輸入「我最近考試壓力很大」。
8. 到節點頁重新連線離線節點。
9. 最後展示匯出、匯入、隱私模式或重置。

## 評審可能會問

### 這是不是在診斷學生心理狀態？

不是。我們只做關懷提醒和支持建議，不做醫療或心理診斷，也不使用真實學生姓名。

### 為什麼要匿名？

因為心理與情緒資料很敏感。展示時用匿名代號，正式使用也應該先保護隱私，再讓老師或輔導室提供支持。

### 沒有 Firebase、Gemini 或 Arduino 可以展示嗎？

可以。系統目前是 local-first，沒有雲端也能完整操作；沒有 Arduino 時會保留硬體提示紀錄，接上 UNO R4 後同一操作會送到實體提示裝置。

## 後續機器人連動計畫

1. 將 `ALERT_SIGNAL` 對應到柔和 LED 提醒，避免造成學生壓力。
2. 將 `CARE_DEPLOYED` 對應到關懷已佈署的提示圖示。
3. 將 `NODE_RESTART` 對應到節點重新連線燈號。
4. 保留 `NODE_HEARTBEAT` 給未來節點定期回報。
5. 堅持不在硬體端顯示真實學生姓名或敏感內容。
